4. Always-Delegate Behavior pending opinion

The Hermes config does not expose a "always try delegate_task first" toggle. Subagent use is a per-decision thing at the LLM level, driven by tool descriptions and the judgment of the agent. To bias the model toward delegation, the practical levers are:

  • SOUL.md instruction: add a line to each profile's SOUL explicitly framing the default ("for any independently-researchable question, default to delegation"). Effect: soft. Model respects but can override.
  • Smart-model-routing for subagents: already done. Cheaper delegation means we're not punished when the model delegates.
  • Skills as soft enforcement: a skill named auto-delegate that loads on every turn and lists delegateable patterns. Effect: medium. Depends on disciplined skill loading.
  • delegation.subagent_auto_approve: true: currently false. Flipping it removes the human-in-loop gate on every subagent call. Effect: high throughput, but loses oversight on cron-driven work. Not recommended until we have validated output quality from the 4B.

Honest read: there is no clean "always delegate" switch. The next step is the test below. If a real research task delegated to the 4B returns usable synthesis, the case for "delegate first, do locally later" writes itself into SOUL.md without needing new config keys.
