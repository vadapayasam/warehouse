PRODUCT 01 · BUILD FIRST

Cascade-Watchdog

A verification layer that locks the chain. Step N fails means step N+1 never starts.

PAIN · FROM LOG
"The processing pipeline broke May 15 and nobody noticed. 14 overnight files generated 13 actionable items, yet Prabha's work... got zero analytical attention." [Synth-2026-06-17]

What it is

A lightweight prove-then-proceed runtime for cron / agent / pipeline orchestration. Wraps any shell script, cron job, or multi-step agent flow and gates each downstream step on a configurable verification predicate of upstream output. Failures short-circuit immediately. Reports land in your inbox, not in a wall of cron log files you never read.

Why it builds itself

Every agent builder hits this. LLM crons, scraping chains, financial data pipelines, the assistant-loop products shipping right now, they all have the silent-brittle failure mode. Nobody is selling the version that is honest about it. You ship a 200-line watcher + a Python SDK, charge $19/mo for hosted, free for self-host. The market is everyone running more than 3 crons.

Stack & shape

runtime: Python, fastapi, sqlite
verify: pluggable predicates (file_exists, n_lines, regex, content_hash, http_2xx)
interface: CLI cw run ./pipeline.sh --gate step2:file_exists=out.json + web hooks
notify: Discord, Slack, email, log-only

PRODUCT 02 · BUILD SECOND

Skill-Health Radar

A heartbeat daemon for your agent's instruction surface. Score every skill/script/tool by use, errors, drift.

PAIN · FROM LOG
17 ghost skills documented in Jun 17 digest. skill_manage errors at 75/day. command-code update channel silently dead. Stale frontmatter nobody noticed.

What it is

Drop-in daemon that instruments every agent skill, command, script, and tool. Tracks activations, error rate, mean latency, last-used timestamp, frontmatter health, and orphan status with the open-design system. Surfaces a daily scorecard: green / yellow / red. Suggests archival candidates. Diff against last week. Warns on silent breakage (the 3 a.m. kind).

Why it builds itself

Anyone with a multi-skill agent stack has invisible rot. The market is small but obsessive: indie hackers running Hermes/OpenClaw-style setups, agent infra teams at startups, the Discord-bots-with-50-commands crowd. You have the live data. Ship a CLI + hosted dashboard. $9/mo per workspace, free for personal.

Stack & shape

runtime: Python, sqlite, optional Next.js frontend
inputs: Hermes session logs, MCP tool calls, command-code history
output: daily scorecard, alert on red drift, weekly digest
diff feature: "your agent's skill stack vs 7 days ago" with a one-click safe-deprecation plan

PRODUCT 03 · BUILD THIRD

Assistant-Context Compactor

The tool the assistant should call at 60% filled, not the emergency fallback you just survived.

PAIN · FROM LOG
Memory limit exceeded 7× in 24h. Two context-summary failures this week (Jun 19 fallback, now). The summarizer-as-finally-fired lifeline is brittle and silent.

What it is

A proactive context compressor the LLM calls unilaterally when its own window is filling. Strict semantic-preservation guarantees (defined cost metrics), deterministic structure for tail recovery, and a degraded-mode contract so the downstream session never sees a gap it didn't plan for. Exposes a clean token-budget API. Targets LangChain / LlamaIndex / custom agent loops.

Why it builds itself

Every agent framework silently trusts that summarization will save them. None of them test the failure. You have stress-tested it (twice, accidentally). Sell the SDK as pip install contextsentry + hosted tier with usage analytics. Indie pro tooling: cheap to build, recurring revenue, durable as long as long-context models stay leaky.

Stack & shape

runtime: Python, pydantic, async http
core: threshold-triggered compaction with verifiable hand-off artifact
guarantees: token-bound, semantic-bound, structured-bound, recovery-bound
monitoring: live token dashboard, cost-per-turn analytics, fail-safe switch
