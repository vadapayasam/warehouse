Local MLX Coder Guide

Use MLX as the small executor

This page is the practical map for the local Qwen Coder endpoint on your Mac Mini. It shows what is wired, what is not, and how to run it in a planner plus executor setup.

OpenClaw MLX-LM Qwen2.5 Coder 3B 4-bit

Current state

Installed

Yes. mlx-lm is installed in ~/.venvs/mlx-lm.

Model

mlx-community/Qwen2.5-Coder-3B-Instruct-4bit is downloaded and serving.

OpenClaw

The local provider is added. Primary is still the cloud model right now.

Is it wired to OpenClaw

Yes, partially.

  • The MLX provider exists in OpenClaw config.
  • The local model is available as a selectable alias.
  • It is not the primary default yet.
  • It is not in the fallback chain yet.

So: connected, but not default and not fallback.

How to use it now

openclaw models list --provider mlx

# route a task to the local model when you want the small executor
# use the alias name you set in config
/model Local MLX Coder

If your session supports model routing, point a subtask at the local alias. Keep the big model for planning.

What it can see

  • Only what OpenClaw sends into the prompt.
  • Recent code or style if you include files, snippets, or repo context.
  • No automatic memory of the whole repo.
  • No internet or web search by itself.

Planner plus executor

Planner

Use the frontier model for reasoning, long context, and tool-heavy decisions.

Executor

Use MLX for short coding jobs, terminal commands, Python, and HTML/CSS/JS edits.

# keep cloud as planner
# route local tasks to the MLX alias

# use the local model for:
# - code cleanup
# - small file edits
# - quick script generation
# - HTML/CSS review
# - command drafting

Online docs and web search

No. The model endpoint alone does not browse the web.

If OpenClaw gives the session browser or web tools, then the agent can fetch docs and search the web. The local model just receives the result text.

Make it fallback later

Right now it is just added. To turn it into a real backup path, move it into the fallback list or switch primary routing depending on your setup.

# idea only, not applied here
# primary = frontier model
# fallbacks = local MLX alias

Good way to run it

  • Cloud model for planning, review, and web-heavy work.
  • MLX for quick local execution and small code tasks.
  • Give the local model only the files it needs.
  • Keep prompts tight so RAM stays low.

Bottom line

Yes, it is installed. Yes, the endpoint works. Yes, OpenClaw sees it. No, it is not the default planner yet. Use it as the small executor, and let the frontier model do the heavy thinking.
