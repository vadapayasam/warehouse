Design Analysis
AI Possible Actions Panel · 2 Variations
Horizontal (A)
Full-width header strip. 4 actions in one row. Fast for experienced technicians in scanning mode. Primary CTA draws eye immediately via filled purple style. Use when: wide layout, quick action expected.
Vertical (B)
Side-panel card. Stacked actions with icon + label + description. Each action explains itself — lower training burden. Recommended for new technicians. Use when: side panel, evaluation mode.
Action Hierarchy
1. Apply Solution — Primary AI path 2. Notify Requester — Communication 3. Update & Log — Documentation 4. Handle Manually — Override (de-emphasized) Override intentionally muted to prevent accidental AI bypass.
Design Decisions
• "Monica suggests" chip = AI attribution • 94% confidence badge = explicit trust signal • Purple = Monica/AI brand consistency • Vertical descriptions = self-documenting UI • Handle Manually always available (never hidden)
