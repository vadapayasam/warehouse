I run a 24/7 AI agent on Telegram. 1,395 sessions last month. 584 million tokens burned. Most of it was wasted. Not on bad prompts. On the agent just... existing. Here's how I cut 30-40% of the bill with 3 config changes.
Every single message you send to your AI agent, the model re-reads: - System prompt (~3,500 tokens) - Agent personality file (~1,900 tokens) - Memory + user profile (~800 tokens) - Tool definitions for every connected service (~7,000 tokens) That's 13,000 tokens before you even type a word. Multiply by 27,000 messages/month. That's your bill.
My agent's personality file (SOUL.md) was 140 lines. Most of it was duplicate sections and meta commentary about "this is a living document." Compressed it to 80 lines. Same personality. Same rules. Just no repetition. Before: 7,495 bytes -> After: 3,641 bytes. Saved ~973 tokens per turn.
"Hey" doesn't need the same processing as "deploy the latest build to production." Enabled smart_model_routing. Messages under 160 characters route to a cheaper model with minimal tools loaded. Simple queries: ~5,000 tokens fixed overhead instead of ~9,400.
My Telegram setup loaded 17 toolsets. Image generation. Text-to-speech. Code execution. Media orchestration. I use Telegram for text input and model switching. That's it. Removed 4 unused toolsets. Saved ~2,000-3,000 tokens per turn. 17 -> 13 toolsets. Zero functionality lost.
Total fixed overhead per turn: Before: ~13,200 tokens After: ~9,400 tokens That's 3,800 tokens saved per message. At 27,000 messages/month, that's ~103 million tokens. Plus I built a shell script library that chains common terminal operations into single commands. No LLM round-trips needed.
Most AI agent optimization advice is about better prompts. But the real token drain is the infrastructure around the prompt. Check your: - Personality files (are they bloated?) - Tool definitions (are you loading services you never use?) - Model routing (are simple queries hitting the frontier model?) The fixes take 20 minutes. The savings compound daily.
