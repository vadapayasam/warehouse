How It Works

YouTube Shorts Algorithm Mechanics

Understanding the recommendation system — session-based distribution, test pools, and the signals that actually matter.

📊

Session-Based Recommendations

Shorts are served in infinite scroll sessions, optimizing for session retention — keeping viewers in the feed. Unlike long-form, there's no search discovery here.

Primary Traffic Source
🧪

SWF Test Pools

Videos are first shown to small test pools (50-500 views). If engagement passes thresholds, the algorithm scales distribution exponentially.

Distribution Mechanism
Signal How It's Measured Target Weight
AVD% Average View Duration Percentage of video watched 80%+ completion Critical
Loop Rate Immediate rewatches 15%+ loop rate High
Engagement Velocity Likes/comments in first 24hrs 5%+ engagement rate Medium
Share Rate Saves/shares per impression 1%+ share rate High
CTR Thumbnail/title clicks 4-10% from browse Medium
💡 Dev-Specific Insight

Developer content often has higher share rates than entertainment because developers save tutorials for later. YouTube treats saves as high-intent signals — more valuable than likes.

Content Models

Build-in-Public Content Patterns

Three proven models — Fireship, Primeagen, and the Indie Dev Playbook — each with different tradeoffs.

🔥
Pattern 1

The Fireship Model

One concept → One code snippet → Immediate takeaway. Information density + shareability. Developers send these to team chats.

  • Length: 15-25 seconds
  • Posting: 1-3x daily
  • Visual: Screen recording + text overlays, no face
  • Content: Language tricks, hidden features, "did you know"
⚡
Pattern 2

The Primeagen Model

Strong opinion → Explanation → Call to action. Comment bait through controversy. Hot takes drive 10x more engagement than tutorials.

  • Length: 30-45 seconds
  • Posting: 5-10x weekly
  • Visual: Face-first + screen B-roll, high energy
  • Content: Hot takes, industry commentary, workflow reveals
🛠️
Pattern 3 — Your Model

The Build-in-Public Playbook

Problem → Build process → Result/Demo. Document the messy middle. Show constraints, tradeoffs, failures.

  • Length: 25-40 seconds
  • Posting: 1-2x daily
  • Visual: Screen recordings + code + Hermes outputs
  • Content: Feature builds, bug fixes, design iterations, real metrics
Attention Capture

5 Proven Hook Formulas

First 3 seconds determine 80%+ of performance. These are the patterns that actually work for dev content.

Formula 1

The Contrarian Hook

[Conventional wisdom] is wrong because [counterintuitive reason]

Example: "Stop using useEffect for data fetching. Here's why 90% of React devs get this wrong."

Formula 2

The Specific Number Hook

I [achieved result] in [time] with [specific thing]

Example: "I shipped 47 app updates in 3 months using this one automation."

Formula 3

The Process Hook

How I [did thing] in [timeframe] — full process

Example: "How I built an iOS widget that tracks my coding time — from idea to App Store in 48 hours."

Formula 4

The Hot Take Hook

[Strong opinion] — and here's why

Example: "SwiftUI is broken for production apps and Apple won't admit it. Here's what they don't tell you."

Formula 5

The Mystery Hook

I discovered [surprising thing] by [unexpected method]

Example: "I discovered YouTube's algorithm favors this ONE format by analyzing 10,000 dev Shorts."

4 Narration Frameworks

Ready-to-use scripts adapted for build-in-public developer content. Modify the specifics, keep the rhythm.

Framework 1: The Contrarian Reveal 
[0-5s] Hook: "Stop doing [common practice]."

Framework 2: The Build Process 
[0-5s] Hook: "Building [feature] for [app]. Here's what I learned"
[5-15s] Constraint: "The challenge: [technical problem]"
[15-28s] Solution walkthrough (show code/process)
[28-35s] Result: Screen recording of working feature
[35-40s] CTA: "Shipped [X] in [time]. What should I build next?"
Framework 3: The Metrics/Update 
[0-5s] Hook: "Day [X] building [app] — here's where we are"
[5-15s] Numbers on screen: downloads, revenue, users
[15-25s] What changed: "Since last update: [specific wins/failures]"
[25-35s] Next move: "This week: focusing on [specific thing]"
[35-40s] CTA: "Subscribe to watch [outcome]"
Framework 4: The Quick Win 
[0-5s] Hook: "Here's the [thing] nobody taught you about [topic]"
[5-12s] Show the thing (code, technique, hidden feature)
[12-20s] Use case: "Perfect for [specific scenario]"
[20-28s] Deeper cut: "And if you [advanced tip], you get [better result]"
[28-34s] CTA: "Save this. You'll need it."
Automation

Hermes Automation Workflow

End-to-end pipeline for creating Shorts using Agent Tina — from raw session to published video.

Step 1

Record Raw Session

  • Screen recording of build/feature demo
  • Vietnam voiceover via iOS Voice Memo
  • Export to Files app, move to Dropbox/whatever
Step 2

Hermes Processing

  • Trigger: Scheduled or manual cron
  • Transcribe with Whisper
  • Extract key moments (auto/Claude)
Step 3

Edit & Package

  • FFmpeg cut to 25-40 seconds
  • Add text overlays/burn captions
  • Export MP4 + thumbnail frame
Step 4

Metadata & Upload

  • Generate title from hook formula
  • Write description + tags
  • Upload to YouTube via API
Cron Config (~/.hermes/config.yaml) 
shorts_pipeline:
  enabled: true
  schedule: "0 18 * * *"  # Daily at 6 PM IST
  raw_path: ~/.hermes/raw/
  output_path: ~/.hermes/output/shorts/
  min_duration: 25
  max_duration: 40
