How I Cut My AI Agent's Token Bill by 30-40%

5 min read

The Problem

I run a 24/7 AI agent on Telegram. It handles my task management, research, iOS builds, web publishing, and terminal operations. Last month, it processed 1,395 sessions, 27,098 messages, and 584,808,288 tokens.

Most of that was wasted. Not on bad prompts. On the agent just existing.

Here's the thing about AI agents: every message you send, the model re-reads the entire system context. System prompt, personality file, memory, user profile, tool definitions for every connected service. That's fixed overhead baked into every single turn.

Before my optimization, my fixed overhead was ~13,200 tokens per turn. Multiply by 27,000 messages/month, and you're burning ~356 million tokens just on context. That's 61% of my total consumption.

The Diagnosis

I dumped the full context breakdown:

System prompt + Hermes base: ~3,500 tokens
SOUL.md (agent personality): ~1,873 tokens
Memory + user profile: ~818 tokens
Tool definitions (Telegram): ~7,000 tokens

The tool definitions were the biggest chunk. My Telegram setup loaded 17 toolsets including image generation, text-to-speech, code execution, and media orchestration. I use Telegram for text input and model switching. That's it.

SOUL.md was bloated. 140 lines of personality rules, but two sections said the same thing. Another section was meta commentary about the document itself.

Fix 1: Compress the Personality File

SOUL.md went from 140 lines to 80 lines. Same personality. Same rules. Just no repetition.

Merged the duplicate pushback sections. Removed the "This Is A Living Document" section (meta, no operational value). Consolidated overlapping workflow rules.

Fix 2: Enable Smart Model Routing

"Hey" doesn't need the same processing as "deploy the latest build to production."

Hermes has a built-in feature called smart_model_routing. It checks message length and routes simple queries to a cheaper model with minimal tools loaded.

Enabled it with a threshold of 160 characters. Messages under that limit route to mimo-v2.5 with reduced tooling. Simple queries now use ~5,000 tokens fixed overhead instead of ~9,400.

Fix 3: Trim Unused Toolsets

Removed 4 toolsets from the Telegram platform configuration:

  • image_gen (I generate images through other channels)
  • tts (I don't use text-to-speech on Telegram)
  • moa (media orchestration, unused)
  • code_execution (terminal handles this)

17 toolsets became 13. Zero functionality lost. Saved ~2,000-3,000 tokens per turn.

Bonus: Shell Script Library

Built tm-scripts, a library of 8 shell scripts that chain common terminal operations into single commands. Research pipelines, deployments, health checks, todoist queries. Each script runs in one terminal call instead of 3-5 LLM round-trips.

Example: tm research "NinjaOne AI copilot MSP" runs SearXNG search, extracts results, and formats output. One command, no LLM tokens burned on intermediate steps.

Takeaway

Most AI agent optimization advice focuses on better prompts. But the real token drain is the infrastructure around the prompt.

Check your:

  • Personality files (are they bloated? duplicate sections?)
  • Tool definitions (are you loading services you never use?)
  • Model routing (are simple queries hitting the frontier model?)

The fixes take 20 minutes. The savings compound daily.
