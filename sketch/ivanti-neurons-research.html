Workflow & Lifecycle

Request Lifecycle

The AI Self-Service Agent handles the full intake-to-resolution pipeline:

  1. Employee types natural language request in chat/portal/Slack/Teams [2]
  2. AI Self-Service Agent interprets intent using NLU + LLM [2]
  3. Agent asks follow-up questions to gather complete context (guided capture) [3]
  4. Agent searches Knowledge Base + curated policies + past tickets [4]
  5. If within policy boundaries, agent executes resolution autonomously [2]
  6. If resolution not possible, agent creates structured incident with full context [6]
  7. AI-assisted triage routes to right team/technician [5]
  8. User confirms resolution; resolution recorded to KB for future use [3]

Agent Capabilities

Ivanti shipped three AI agents in Q1-Q2 2026, all running on the same LLM backend (undisclosed model, likely GPT-4 or fine-tuned variant) [2].

Self-Service Agent

  • Conversational front door for IT support [2]
  • Natural language intake and guided troubleshooting
  • Autonomous resolution of common issues
  • Multi-step planning across KB + policies [3]
  • Knowledge grounded in CMDB + past tickets [4]

Incident Creation Agent

  • Converts natural language to structured incidents [1]
  • Auto-populates priority, category, impact [3]
  • Infers urgency from conversation context
  • Integrates with ITSM data model [1]

Service Request Agent

  • Handles catalog requests (software, access, hardware) [6]
  • Validates requestor eligibility [6]
  • Checks asset availability via ITAM
  • Routes approval chain + triggers fulfillment
  • Pulls HR context for user attributes [4]
Source: Ivanti agentic AI page [1], Q2 2026 release notes [6]

Limitations & Gaps

  • No PSA integration - No native ConnectWise/Autotask/Kaseya connectors [7]
  • No MSP billing - Quote-based enterprise licensing, no per-client billing [7]
  • No multi-tenancy for MSPs - Each client needs separate instance [7]
  • AI pricing unclear - AI Self-Service Agent pricing not published as of June 2026 [7]
  • Closed ecosystem - AI agents only work within Ivanti Neurons [2]
  • No MCP protocol support - No agent interoperability [1]

MSP Gap Summary

Ivanti Neurons is enterprise ITSM with AI agents, not an MSP platform [7]. These gaps make it a poor fit for most MSPs:

1
No PSA integration (ConnectWise/Autotask/Kaseya) [7]
2
No multi-tenant architecture - each client needs separate instance [7]
3
Quote-based pricing incompatible with per-client billing [7]
4
AI add-on costs unknown - can't factor into service pricing [7]
5
No MCP/agent interoperability [1]
6
Heavy deployment - requires full Neurons platform [4]

Bottom Line

Skip unless you are already a full Ivanti shop [7]. The agentic AI is impressive but locked inside an enterprise-priced, single-tenant platform that doesn't fit MSP operational models [7].
