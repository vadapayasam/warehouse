Verdict

Three models, three roles. Here's when to use each.

Qwen2.5-Coder-3B

Daily Driver · King

Use for everything utility: codegen, HTML, summarization, filtering, CSS, regex, bash. Fastest, cleanest, lowest RAM, zero overhead. Not designed for agentic tasks.

Qwen3.5-4B-OptiQ

Agentic Complement

Use when you need agentic capability: multi-step tool plans, JSON output, structured formats. Thinking OFF. 2x slower, heavier RAM. Stronger output quality on prose/summary.

Qwen3.5-4B+Tooling

Tool Calling Workhorse

Same Qwen3.5-4B engine with RapidMLX tool pipeline enabled. 5/5 across all agentic benchmarks — JSON extraction, multi-step planning (both tools), multi-turn chaining (parallel calls), self-directed selection, and clean native tool_calls format. Three flags: --tool-call-parser hermes, --enable-auto-tool-choice, --enable-tool-logits-bias. 3.4 GB RAM, ~20 tok/s.

LFM2.5-8B-A1B

Specialist · Benchwarmer

Use only when you need chain-of-thought reasoning and can spare 4096 tokens. Best RAM efficiency for its size. Useless for utility tasks under 1024 budget. Keep for specialist use.

This Is the Way

3B Coder stays king. Qwen3.5-4B stays as agentic complement. Qwen3.5-4B+Tooling replaces Qwen3-8B as the tool-calling workhorse — same model, same RAM, same speed, but with RapidMLX's full tool pipeline enabled. The three flags solve format drift, multi-step inconsistency, and self-directed routing problems. LFM8B stays as specialist depth. The hierarchy grows, no one replaced.
