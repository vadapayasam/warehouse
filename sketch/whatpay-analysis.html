Screen 1: Onboarding — The Trust Problem

Visual Design

  • • Pure black (#000000) background — OLED optimized
  • • White text — high contrast meeting WCAG AAA
  • • iA Writer Quattro font with SF fallback
  • • "Get started" inverted pill CTA
  • • Credit card icon in dark gray circle

Interaction

  • ✓ Multiple opt-outs (Skip, Remind me later)
  • ✓ Clear value proposition
  • ✗ No progress indicator (1 of 3?)
  • ✗ Low contrast on secondary CTAs
  • ✗ Zero trust signals for financial app

Critical Issue: Trust Deficit

For an app handling subscription/financial data, this onboarding has zero security messaging. No "Bank-level security," no "Your data stays on device," no privacy badges. Users abandon financial apps here.

Fix This Week

  • → Add trust badge: "Your data stays on your device. No sign-up required."
  • → Lighten "Remind me later" to #B0B0B0 (WCAG AA compliance)
  • → Add progress dots if expanding to multi-screen

Gap Analysis

Critical

No Trust Signals

Financial app onboarding lacks any security messaging. Users don't know if data is local-only or synced. Add "Your data stays on your device" badge.

Critical

No Import from Competitors

Users coming from Bobby, Truebill, Copilot can't migrate. Build CSV import — competitors export JSON/CSV.

Medium

No Smart Defaults

Type "Netflix" → should auto-select Streaming. Type "Spotify" → Music. Implement keyword → category mapping.

Medium

Onboarding is Generic

Single screen with stock credit card icon. Expand to 3-step: Value → Permissions → Setup with personality.

Nice

No Savings Calculator

Show "Switch to annual → Save $X" per subscription. This is table stakes for subscription managers.
