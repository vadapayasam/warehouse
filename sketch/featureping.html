FeaturePing

Turn product release notes into in-app explanations users actually understand. A narrow widget for teams that ship often and need users to notice, try, and adopt new features without building a full onboarding suite.

The Problem

Changelogs are cemeteries.

SaaS teams ship faster than users absorb. Release notes live outside the product. Emails get ignored. Slack posts scroll away. Support repeats the same "where is this?" explanation.

The existing onboarding market answers this with broad suites: tours, checklists, surveys, NPS, session replay, resource centers, and enterprise procurement. Small teams do not need another platform. They need a lightweight loop:

Product update → in-app explanation → "Show me" action.

That is the entire job FeaturePing targets.

The Wedge

Not an onboarding platform. A feature explainer.

FeaturePing converts a rough release note into short user-facing copy, an in-app announcement, and a "Show me" button that navigates to and highlights the changed UI element. No full tours. No analytics dashboards. No enterprise procurement.

1. Paste the note

Drop a raw release note or changelog entry into the composer. AI rewrites it into announcement, tooltip, why-it-matters, and help text.

2. Pick the target

Set the page URL and optional DOM selector. The widget knows where the feature lives and can navigate to it cross-page.

3. Embed the widget

A single async JS snippet renders the "What is New" feed. Users click "Show me" and the widget scrolls, highlights, and points.

4. Export everything

Markdown and JSON exports keep updates portable. No lock-in. No migration tax if you outgrow the tool.

Architecture

Four layers. TypeScript everywhere. JSON-first.

1. Admin Composer

Web dashboard for creating updates, pasting raw notes, choosing tone, setting target URL and DOM selector, and previewing widget output.

2. AI Copy Generator

Converts raw release notes into announcement, why-it-matters, tooltip, and help snippet. Hard rule: no invention. Missing context is flagged, not fabricated.

3. Widget Runtime

Vanilla TypeScript, framework-agnostic, async-loaded. Fetches active updates, filters by URL and audience, renders feed or modal, handles dismiss state, and executes "Show me" navigation and highlight.

4. Event + Export

Lightweight counters only: views, dismisses, show-me clicks. Exports to Markdown and JSON. No deep analytics in MVP.

Data model

The Update entity is the core contract:

  • id, product_id, title, raw_note
  • status: draft | published | archived
  • target_url and optional target_selector
  • audience: plans, roles
  • copy: announcement, why_it_matters, tooltip, help_snippet
  • created_at, published_at
