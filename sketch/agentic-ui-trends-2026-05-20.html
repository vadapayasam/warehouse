Agentic UI Trends

What interfaces for autonomous software actually look like in 2025. Not predictions. Patterns that are shipping now.

20 May 2026 · Daily Research

Generative UI

Components that materialize mid-conversation

Vercel's AI SDK pushed streamUI() and createStreamableUI() into production this year. The idea: the LLM doesn't just return text, it returns React components directly into the chat stream. You ask about the weather, you get a live weather card. You ask about stock prices, you get an interactive chart. Not a link to a dashboard — the dashboard is the reply.

This kills the chatbot ceiling where everything degrades to text walls. The trade-off: you need a typed component library on the server, and the LLM needs to know what components exist. But once wired, the conversation becomes the application.

Source: Vercel AI SDK — Generative User Interfaces

Reasoning UI

Showing the work

OpenAI's o3/o4-mini and Anthropic's extended thinking models output reasoning chains before final answers. The UI pattern that stuck: a collapsible "Thinking" block nested under the assistant bubble, streaming token-by-token in muted gray. Users can expand to audit the logic or collapse to skip to the answer.

This isn't a debug feature. It's a trust feature. When an agent recommends a $50K infrastructure change, the user wants to see how it got there. The design problem: reasoning chains can be long and noisy. The good implementations truncate gracefully and let power users expand.

Sources: OpenAI o3/o4-mini, Anthropic — Building Effective Agents

Tool-Use Cards

Making the invisible visible

Anthropic's MCP and Claude Code expose tool calls as inline cards in the conversation. The agent calls a file search, a card appears: "Searching files for 'auth middleware'" with a spinner. When done, the card flips to show results. Destructive actions trigger a permission toast: "Allow edit to package.json?"

Before this, tool calls were invisible. The agent would pause mysteriously, then return with an answer. Users had no idea if it was thinking, broken, or calling an API. Tool cards make the agent's hands visible.

Source: Anthropic — Building Effective Agents

Permission Modes

A dial, not a switch

Claude Code offers three tiers: Ask (every tool needs approval), Auto-edit (file writes ok, shell commands ask), and Auto (full autonomy with an audit log side panel). The insight is that autonomy isn't binary. Users want different trust levels for different tasks — refactoring a test file vs. running rm -rf /.

The audit log is the critical companion. Full auto without rewind is reckless. The UI needs a time-travel panel showing every action taken, with one-click undo.

Source: Claude Code Best Practices

Multi-Agent Orchestration Dashboards

When one brain isn't enough

Anthropic's multi-agent research system uses a lead orchestrator that spawns parallel subagents. The UI shows a tree: root query at top, branches for each sub-agent, status badges (running / done / failed), token burn per branch. The final answer synthesizes outputs from all branches.

Single-context agents choke on breadth-first tasks like "list all board members of S&P 500 tech companies." Parallel agents solve this, but without visualization, the user has no idea what's happening. The dashboard makes parallel execution legible.

Source: Anthropic — Multi-Agent Research System

Workflow Visualization

Code as a map

Vercel's AI SDK supports directed graphs for agent logic: prompt-chaining, routing, parallelization. The emerging UI pattern: render these as node-edge canvases where each node is an LLM call or tool. Users can trace why the router classified their intent as "billing" instead of "support."

This is less about aesthetics and more about debugging. When an agent takes a wrong turn, the map shows exactly where the decision happened. It's the stack trace for agent logic.

Source: Vercel AI SDK — Agents

Agentic Infrastructure Observability

Monitoring agents like services

Vercel's "Agentic Infrastructure" describes dashboards for agent workloads: traces of model calls, tool invocations, retries. Cost and token-burn graphs per deployment. Sandboxed execution logs. Human-approval queues for autonomous actions.

The shift: agents aren't just users of infrastructure, they're creators of it. An agent deploys code, then the infrastructure UI must let humans audit, approve, and roll back those deployments. The platform becomes a human-in-the-loop agent itself.

Source: Vercel — Agentic Infrastructure

Suggestion Chips

Steering without typing

When agents hit ambiguity, the UI surfaces quick-reply actions: "Compare plans", "Show pricing", "Talk to sales." This pattern, common in ChatGPT and Copilot, reduces cognitive load. The user doesn't need to formulate a follow-up question — they tap a chip.

The design risk: too many chips feel like a decision tree from 2005. The good implementations show 2-3 contextually relevant options, not a grid of every possible action.

Context Source Indicators

What the agent actually knows

MCP-connected agents pull from files, APIs, and databases. The UI needs to show which sources are loaded. A subtle indicator — "Reading from 3 files" with an expandable list — tells the user whether the answer is grounded in their actual data or hallucinated from training.

This is the citation bar for the agentic era. Without it, users can't distinguish between "the agent read your codebase" and "the agent made a plausible guess."

Declarative Agent Builder

No-code agent creation

Microsoft's Copilot Studio lets non-developers build agents via form UIs: pick knowledge sources, define trigger phrases, set conversational topics. The agent compiles to a JSON manifest. An evaluation pane tests against sample utterances before publish.

This democratizes agent creation but creates a governance problem: who approves the HR bot that has access to payroll data? The UI needs admin controls alongside the builder.

Source: Microsoft 365 Copilot Extensibility

What this means for designers

The agentic era isn't just chatbots with better CSS. It's a new interaction paradigm where conversations generate interfaces, trust requires transparency, autonomy needs guardrails, parallel work needs visualization, and debugging needs maps.

The designers who get this aren't the ones adding AI sparkle to existing UIs. They're the ones rethinking what a user interface is when the software thinks for itself.
