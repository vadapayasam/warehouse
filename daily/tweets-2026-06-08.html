1 #hermes listicle 10:30 AM IST 
Why I built my own agent system instead of using ChatGPT:

1. Persistent memory across sessions
   ChatGPT forgets everything when the conversation ends. My agent remembers my project structure, my preferences, my past failures. Every session starts with context, not from zero.

   Real example: I asked Tina to "push WhatPay to TestFlight" yesterday. Today I said "do it again" and she already knew the scheme, bundle ID, and API key path. No re-explaining. No re-configuring.

2. Custom skills for my exact workflow
   Not generic "help me write code." Specific: "build WhatPay for TestFlight using my signing config." The skill knows my scheme, my bundle ID, my API key path.

   I have 30 skills now. Five get used daily. The rest are there when I need them. That's the right ratio. Skills are not Pokemon. You don't collect them. You weaponize the ones that remove friction.

3. Cron scheduler for recurring tasks
   Monday 9 AM: ITSM research runs automatically. I wake up to a digest. Not a reminder to do research. The research is already done.

   My crons: daily tweets at 6 AM, weekly ITSM research at 9 AM Monday, daily Todoist processing at 6 PM, overnight research at 2 AM Wednesday, overnight competitor scan at 2 AM Friday.

   I do not touch any of these. They run, they write, they commit, they push. I review the output when I feel like it.

4. No $20/month subscription
   DeepSeek v4 Pro via Ollama: $0. Claude via API: pay per use, not flat fee. Total monthly cost: under $5 for my volume.

   Breakdown: Claude API ~$3/month for the heavy lifting. Ollama for daily driver. Firecrawl self-hosted for research scraping. SearXNG self-hosted for search. The only recurring cost is electricity.

5. My data stays local
   No training on my code. No retention of my financial data. My vault is on my disk. My memory is on my machine.

   I committed API keys to git once. The agent caught it before push. That's the kind of safety you only get when the system knows your history.

Total cost: ~$5/month. Total control: 100%.

3 #tips single 18:00 IST 
Pro tip: Keep a `.gitignore` template for iOS projects.

I accidentally committed build artifacts, .asc/ files, and DerivedData into WhatPay repo. 47 files. The repo ballooned from 2 MB to 87 MB. GitHub was not happy.

The fix was not just `git rm -r --cached`. I had to rewrite history because those binaries were in the commit chain. I used `git filter-repo` (after installing it via pip because it's not in Homebrew by default).

One `git rm -r --cached` + proper `.gitignore` = clean repo forever.

My iOS `.gitignore` template now lives at `~/.gitignore-ios` and I copy it into every new project:

- `build/`
- `DerivedData/`
- `*.xcuserstate`
- `*.xcworkspace/xcuserdata/`
- `.asc/` (App Store Connect CLI cache)
- `*.ipa`
- `*.app.dSYM.zip`

Also: add `*.env` and `*.pem` even if you think you'll never commit them. You will. At 2 AM. When you're tired.

Working on TimeLeft this week.

SwiftUI + WidgetKit combo is still undefeated for quick utility apps. Here's why, with actual build times:

Live Activities: Lock screen widget shows remaining leave days. No app open needed. This is the "always visible" feature that makes a utility app stick.

Implementation: 12 lines of `WidgetBundle` + `ActivityConfiguration`. The hard part was not the code. It was understanding the timeline provider refresh logic. Apple docs are fragmented. I found the answer in a Reddit thread, not developer.apple.com.

SwiftUI List + Form: Built the entire settings screen in 40 lines. No custom UI. No Auto Layout wrestling. System handles dark mode, dynamic type, accessibility.

App Intents: Siri can add a leave day. "Hey Siri, I'm taking Friday off." Integration took 2 hours. Not because it's complex. Because Apple made the API actually usable for once.

WidgetKit timeline: Widget refreshes at midnight automatically. No background fetch. No push notifications. System updates the view when the date changes.

Total build time: 3 evenings. ~8 hours. App Store submission pending. That's the power of staying inside the system instead of fighting it.

8 #failures story 14:00 IST 
My Honcho memory system hit capacity. Context windows were getting compressed. Facts were being dropped.

The symptoms:
- Agent started repeating questions I had answered before
- Cross-references in the wiki were getting stale. Links pointed to files that no longer existed
- The daily digest was missing items that should have been tracked. I found 3 research leads buried in raw files that never got processed

I checked the memory store: 2,147 entries. The system was deduplicating but the dedup algorithm was too conservative. It merged exact duplicates but missed near-duplicates like "Use DeepSeek v4 Pro" and "DeepSeek v4 Pro via Ollama is the daily driver."

The fix was "goblin surgery" - my term for aggressive manual cleanup:

1. Deduplicated entries: 200+ near-duplicate facts merged
2. Pruned stale facts: removed 180 entries older than 6 months with no recent references
3. Reorganized structure: split the monolithic memory file into category-specific files (ios, design, workflow, failures, tools)
4. Added a TTL policy: facts auto-expire after 12 months unless renewed by reference

Result: 1,400 entries. Faster lookups. Better context retention. The agent no longer asks me what my bundle ID is.

The lesson: persistent memory needs maintenance. Left alone, it becomes a hoarder's attic. You keep adding but never removing. Eventually the system chokes.

Sometimes you need to delete to grow.
