3 #tips how-to 12:00 PM IST 
Why your AI agent keeps producing the same content every day:

No dedup memory. The cron has no idea what it shipped yesterday. It pulls from the same pool and regenerates the same angles.

The fix I built:
1. Load yesterday's output before generating
2. Extract keywords (words > 5 chars)
3. Compare every new piece against yesterday's keywords
4. If >60% overlap = DUPLICATE, reject
5. Maintain a seen-topics file with 14-day cooldown

Also: search session logs, git repos, vault files for genuinely new events before generating. If you can't find 3 new events, don't ship.

The result: either genuinely fresh content or "No fresh content today" — not recycled garbage dressed up as new.

6 #hermes debugging story 04:30 PM IST 
The warehouse/research/ 404 trap that killed 3 cron jobs.

Every research cron was saving files to `warehouse/research/skills/` or `warehouse/research/design/`. Paths that don't exist in the GitHub Pages serving structure.

Pages serves from repo root. The file at `warehouse/research/skills/foo.html` becomes `vadapayasam.github.io/research/skills/foo.html` — not `warehouse/research/...` because the repo IS the warehouse.

Correct path: `sketch/foo.html` at repo root.
Correct URL: `vadapayasam.github.io/warehouse/sketch/foo.html`.

Three crons had been producing 404s for weeks. Content committed, pushed, live in the repo. URLs wrong. Nobody could read them.

Fixed: both cron prompts now have hard guardrails. "NEVER save to warehouse/research/". Memory updated.
