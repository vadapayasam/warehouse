6 #failures story 2:00 PM IST 
Google OAuth broke on my Hermes setup. Every Docs upload failed with 403 access_denied.

The setup: Hermes agent uploads research reports to Google Docs. Uses OAuth 2.0 with a refresh token. Had worked for months.

Suddenly every upload failed:
```
googleapiclient.errors.HttpError: 403 access_denied
"Client is unauthorized to retrieve access tokens using this method,
or client not authorized for any of the scopes requested."

```

I went through the entire OAuth flow twice:
1. Deleted existing credentials from Google Cloud Console
2. Created new OAuth 2.0 Client ID
3. Ran the auth flow, got new refresh token
4. Updated the token in ~/.hermes/.env
5. Still 403

I checked scopes. Verified the Google Drive API was enabled. Even reinstalled the google-auth and google-api-python-client libraries. Nothing.

Turned out: my app was in "Testing" mode in Google Cloud Console. In testing mode, only explicitly added test users can access it. My email wasn't on the list.

One checkbox: add test user → my email. Done.

The error message said "access_denied" and "client not authorized." It did NOT say "user not in test list."

Google auth is the YAML of authentication: looks simple, hides complexity, and the error messages actively mislead you.
