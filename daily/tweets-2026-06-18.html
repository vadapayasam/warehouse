2 #interesting 12:00 PM IST 
RapidMLX went live on my Mac Mini yesterday for local inference. 4 failed approaches before the 5th worked.

Failed 1: Plain-text JSON prompts. 40% pass rate. The 4-bit model hallucinates structure without a schema.

Failed 2: Skipping --no-thinking. 50-72% token waste on mandatory reasoning that doesn't improve output.

Failed 3: Non-API tool calling through Hermes parser. Broken. The parser can't reconstruct the corrupted output.

Failed 4: Missing vision support. Qwen3.5-4B is text-only. The 2B vision model needs a separate install.

What worked: OpenAI tools parameter + --no-thinking + persistent server on port 8765. 100% pass rate. 0 frontier tokens per call. ~1.7GB RAM pinned.

Verdict If you're setting up local inference on Apple Silicon, skip the trial-and-error. Exact working config: `rapid-mlx serve --model qwen3.5-4b-4bit --no-thinking --port 8765` + OpenAI tools parameter in your client. No plain-text prompts, no parser tricks. The writeup is on GitHub Pages if you want the full failure log.
3 #hermes 4:00 PM IST 
My daily tweet generator cron was paused for so long that the prompt itself became a placeholder: "[PAUSED] Daily tweet generator -- paused by request. No action needed."

When I asked Tina to run it manually today, the cron fired, did nothing, and re-enabled itself. Had to pause it again.

Lesson: a paused cron with a dead prompt isn't paused -- it's orphaned. The system treats it as a live job with no instructions. Better to either delete it or give it a real prompt that exits early when there's nothing to do.
